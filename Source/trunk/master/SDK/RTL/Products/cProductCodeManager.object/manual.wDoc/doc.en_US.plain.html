Retail product code manager

Every product can have many different codes including EAN, UPC etc. but only one code per type. So, a product can have only one EAN code and one UPC. Product codes are being used only as reference and can change any time. However, a code is considered the identity of the product, so changing a code should happen only when mistakes happen.

// Use classes
use \RTL\Products\cProduct;
use \RTL\Products\cProductCodeManager;

// Get product id
$productID = engine::getVar("id");

// Create an empty instance
$codeManager = new cProductCodeManager($productID);

// Set a new product code
$codeManager->set($productCodeType = 1, $productCode = "446516543126542", $expirationTime = "");


// To update a product code, we will call the same function as above
// This will see that there is a code of the same type and it will update
$codeManager->set($productCodeType = 1, $productCode = "446516543126742", $expirationTime = "");


// Remove a code from the database
// This action cannot be undone
$codeManager->remove($productCodeType = 2);

Select a product code type

When creating or updating a product code, we need to provide a code type. This class can provide us with a list of all code types that the SDK supports:

// Use classes
use \RTL\Products\cProductCodeManager;

// Get all code types
$types = new cProductCodeManager::getCodeTypes();

// The above should return something like this:
// It is: key => title
Array
(
    [0] => "EAN"
    [1] => "UPC"
    [2] => "ISBN"
)
