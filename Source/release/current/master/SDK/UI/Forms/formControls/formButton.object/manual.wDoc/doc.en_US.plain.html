Platform Generic Form Button

Developers can create specific form buttons to match the platform or create their own buttons using formItem (or formInput). formbutton is an internal object that is being used by the form builder in order to have less, more flexible and smarter code.

The formButton extends the formItem which minimizes the code even more, allowing the platform to be faster and more efficient. Example:

// Single-line creator of a form button
	
// Create a form button
$fb = new formButton();

// This is a UIObject (extends formItem), so we build
// We use the chain pattern to get the DOMElement immediately
// The type must be a valid button type, otherwise it will return NULL object
$button = $fi->build($title = "Submit", $type = "submit", $id = "", $name = "", $positive = FALSE)->get();

// Append input to form inner container
DOM::append($formContainer, $button);


// We can also create buttons that look positive, like OK, Save, Execute
// While buttons like NO, Cancel, Reset look like simple buttons
$button = $fi->build($title = "Submit", $type = "submit", $id = "", $name = "", $positive = TRUE)->get();

// Append input to form inner container
DOM::append($formContainer, $button);

Javascript interface + examples

formButton also provides a javascript interface where a developer can create form buttons on the client side dynamically.

// Create a form button in a single line
// formButton.function(title, type, id, name, positive);
var fb = formButton.function("Submit", "submit", "button_id", "", true);

// Append to form
jq("form.userform").append(fb);
