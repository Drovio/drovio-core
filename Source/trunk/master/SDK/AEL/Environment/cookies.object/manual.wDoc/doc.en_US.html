Application cookies

Every application has absolute access to the browser of the user and can interact with any way possible. It can also store cookies using this interface.

Cookies are specific storage units that are stored in the browser. This class has access to the cookies that are sent through requests to the server, therefore when a cookie is created, it will be available from the next request.

PHP Example

// Store cookies and read them later
	
// Use the class
use \AEL\Environment\cookies;

// Create a cookie
// This will create an application specific cookie
cookies::set($name = "user_id", $value = "1", $expiration = 0, $httpOnly = FALSE, $secure = FALSE);

// On another request we can read the cookie
$userID = cookies::get($name = "user_id");

// Or even delete it
cookies::remove($name = "user_id");

Javascript interface + examples

There is also an easy way to read and write cookies from javascript side of the application, using a ready interface. Using javascript cookies are instantly available using this interface.

// Read a cookie
Function:	get()
Description:	Read a cookie value
Usage:		appcookies.get(c_name);

// Set a cookie
Function:	set()
Description:	Set a cookie value
Usage:		appcookies.set(c_name, value, exdays, path);
Example:	appcookies.set("user_id", 1, 0, "/");

// Example to remove a cookie:
appcookies.set("user_id", 1, -1, "/");

NOTE: Keep in mind the cookies' restrictions based on the httpOnly and secure parameters.
