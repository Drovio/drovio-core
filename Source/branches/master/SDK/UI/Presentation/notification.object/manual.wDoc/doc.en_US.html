System Notification Object

Notifications is one of the most important components in the system. We can see them pretty much everywhere there is interaction. On every form, error or success a notification of some kind will appear.

Developers are allowed to create a notification in four different types: success, error, warning and info. And they can customize the content as much as they want in order to have the desired outcome.

// Create a simple success notification
	
// Use the class
use \UI\Presentation\notification;	
	
// Create the notification object
$ntf = new notification();

// This is a UIObject, so we build
// There is the chain pattern, so we get the DOMElement immediately
$notification = $ntf->build($type = notification::SUCCESS, $header = TRUE, $timeout = FALSE, $disposable = FALSE)->get();

// Append the notification to the container
DOM::append($container, $notification);

Custom notification messages

You can append custom messages in the notification by calling the append() or the appendCustomMessage() function. The later function will create a small message which is considered just a note for the notification itself. Example:

// Create an error notification with an error message
	
// Use the classes needed
use \UI\Presentation\notification;
	
// Create the notification object
$ntf = new notification();

// This is a UIObject, so we build
// There is the chain pattern, so we get the DOMElement immediately
$notification = $ntf->build($type = notification::ERROR, $header = TRUE, $timeout = FALSE, $disposable = TRUE)->get();


// Append a custom header and a message
$header = HTML::h2("Something went wrong");
$notification->append($header);

$message = HTML::p("An error occurred while trying to execute this function.");
$notification->append($message);

// The custom message for extra information
$notification->appendCustomMessage("Notification Name: getCustomNotificationMessage()");

// Append the notification to the container
DOM::append($container, $notification);

System notification messages

Using the getMessage() function you can get pre-defined notification messages from the system. This is the list of all messages supported at the moment: (messages can change from time to time, these are the descriptions and not the actual messages)

Type Name Description
notification::WARNING wrn.content_uc A specific page/content is currently under construction.
notification::WARNING wrn.function_uc A specific service is currently under construction.
notification::ERROR err.access_denied Insufficient access rights.
notification::ERROR err.authenticate Account authentication error.
notification::ERROR err.invalidate The posting form has been invalidated.
notification::ERROR err.invalid_access Invalid access of our services.
notification::ERROR err.invalid_data Invalid information during form post.
notification::ERROR err.save_error An error occurred while trying to save content.
notification::ERROR err.sys_error An system error occurred.
notification::SUCCESS success.save_success The content saved successfully.
notification::DEBUG dbg.content_not_found The content requested cannot be found (file or something).
notification::DEBUG dbg.exception An exception was thrown.
