Relations person mail manager

Every relation person can have 0 or more email addresses and they can by of different type (home, work etc.). You can add mail addresses to a person as easy as two lines of code and you can edit and remove them even easier:

// Use classes
use \ENP\Relations\ePerson;
use \ENP\Relations\ePersonMail;

// Get person id
$personID = engine::getVar("id");

// Create an empty mail instance
$pMail = new ePersonMail($personID);

// Create new mail (work, home etc.)
$pMail->create($typeID = 1, $mailAddress = "john_doe@unknown.com");


// Update the mail info
$pMail->update($typeID = 1, $mailAddress = "john_doe@example.com")

Getting all mail addresses

// Use classes
use \ENP\Relations\ePerson;
use \ENP\Relations\ePersonMail;

// Get person id
$personID = engine::getVar("id");

// Get all person mail addresses
$pMail = new ePersonMail($personID);
$allMail = $pMail->getAllMail();

// The above should return something like this:
Array
(
    [0] => Array
        (
            [id] => "1_1_1"
            [person_id] => "1"
            [mail_id] => "1"
            [type_id] => "1"
            [mail] => "john_doe@example.com"
            [type] => "Home"
        )
)

Select a mail type

When creating or updating a mail address, we need to provide an address type. This class can provide us with a list of all address types that the SDK supports:

// Use classes
use \ENP\Relations\ePersonMail;

// Get all mail address types
$types = new ePersonMail::getMailTypes();

// The above should return something like this:
// It is: key => title
Array
(
    [0] => "Home"
    [1] => "Work"
    [2] => "Fax"
)
