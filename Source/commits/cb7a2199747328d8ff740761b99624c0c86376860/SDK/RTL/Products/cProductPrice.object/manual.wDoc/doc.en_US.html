Retail product price manager

Every product can have many different prices like retail, wholesale or any company custom price but only one price per type. Price values are used only for reference and they can change any time without affecting other tables. Prices are using the company currency as set from the company financial information.

// Use classes
use \RTL\Products\cProduct;
use \RTL\Products\cProductPrice;

// Get product id
$productID = engine::getVar("id");

// Create an empty instance
$priceManager = new cProductCodeManager($productID);

// Set a new product price
$priceManager->set($priceType = 1, $price = "19.95");


// To update a product price, we will call the same function as above
// This will see that there is a code of the same type and it will update
$priceManager->set($priceType = 1, $price = "15.95");


// Remove a price from the database
// This action cannot be undone
$priceManager->remove($priceType = 2);

Select a price type

When creating or updating a product price, we need to provide a price type. This class can provide us with a list of all price types that the SDK supports:

// Use classes
use \RTL\Products\cProductPrice;

// Get all price types
$types = new cProductPrice::getPriceTypes();

// The above should return something like this:
// It is: key => title
Array
(
    [0] => "Retail"
    [1] => "Wholesale"
    [2] => "Buyer"
)
