Form Notification Object

When we use async forms, there is the need to create specific notifications with the ability to be returned in form report directly. For this reason we created form specific notifications that are equipped with report handlers. They extend the normal system notifications.

Examples:

// Create a form successful notification in response of a form post
	
// Use the class
use \UI\Forms\formReport\formNotification;
	
// Create the notification object
$ntf = new formNotification();

// This is a UIObject, so we build
// We do not have to get the DOMElement in this case, just build
$ntf->build($type = notification::SUCCESS, $header = TRUE, $timeout = FALSE, $disposable = TRUE);

// Get the report directly to the client
// We can reset the form (reset input fields to initial state, if the notification is success) or not
return $ntf->getReport($fullReset = TRUE, $holder = NULL);

Actions and extra content

This form notification object is a way of responding to the client with information. For this reason this object has access to add extra content asynchronous and set actions (trigger events):

// Create a form successful notification in response of a form post with an extra content and an action
	
// Use the class
use \UI\Forms\formReport\formNotification;
	
// Create the notification object and build
$ntf = new formNotification();
$ntf->build($type = notification::SUCCESS, $header = TRUE, $timeout = FALSE, $disposable = TRUE);


// Before getting the report, let's add an extra paragraph somewhere
$p = DOM::create("p", "This is an extra paragraph");
$ntf->addReportContent($p, $holder = ".paragraph-holder", $method = "append");


// Also add an action to refresh something
$ntf->addReportAction($name = "users.list.refresh");


// Get the report directly to the client
// We can reset the form (reset input fields to initial state, if the notification is success) or not
return $ntf->getReport($fullReset = TRUE, $holder = NULL);
