Application logging

Although the platform does not have a debugging environment with direct access to the server, there are tools that help log and debug messages whithout interrupting the normal flow of the application. So, instead of using the network in developer tools and watching the output of the view (echoing the debug messages), you can use logger to log messages and access them through the developer control panel on the toolbar.

Usage Example

// Use the class
use \AEL\Profiler\logger;

// Log a simple message
logger::getInstance()->log("This is a simple debug log message", $level = logger::DEBUG);

// We can create more permanent logs and even write them to file logs (this is done automatically above one level)
logger::getInstance()->log("The view will throw this exception because something went really wrong!", $level = logger::ERROR);
