Relations person manager

This is the basic class for managing team relations, and more specific relation persons. It provide an interface to the Relations SDK for creating, updating and removing relations for the current team.

Creating a new person

// Use the class
use \ENP\Relations\ePerson;

// Create an empty instance
$person = new ePerson();

// Create a new person
$status = $person->create($firstname = "John", $lastname = "Doe", $middlename = "Will");

Update information and remove persons

Using the relation SDK you have full access to the relations of the team, allowing to edit and remove persons as well:

// Use the class
use \ENP\Relations\ePerson;

// Create an existing person instance
$person = new ePerson("1_1");

// Update information
$person->update($firstname = "Jane", $lastname = "Doe", $middlename = "Maria", $notes = "This is a test note");


// And if we want to remove the person, we simply write:
// This action cannot be undone
$person->remove();

Connection to platform users

Relation persons can be "connected" to platform users using their email. Using the getDrovioPersonsByMail() function you can get a list of all the users that the current person matches given their email address.

// Get all matched persons
// This function will return an array of user person ids (not accounts)
$persons = $person->getDrovioPersonsByMail();

// You can connect to the first person
if (!empty($persons))
{
	// Connect to the first person id
	$personID = $persons[0];
	$status = $person->connectToPerson($personID);
}

// Now get the connected person information
$connectedPersonInfo = $person->getConnectedPersonInfo();
